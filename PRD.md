# FluidMusic — 产品需求文档 (PRD) v1.0

---

## 1. 项目概述

### 1.1 项目名称
**FluidMusic** — 流体动态音乐播放器

### 1.2 项目定位
一款支持多平台音乐授权登录、以3D粒子+流体物理为核心视觉语言的桌面音乐播放器。整体UI采用透明流体力学动态背景，所有交互元素以"悬浮气泡仓"形式环绕中心核心布局，营造3D环绕意境。

### 1.3 技术栈
| 层级 | 技术 | 说明 |
|------|------|------|
| 桌面框架 | Electron 33+ | macOS 优先 |
| 3D渲染 | Three.js (WebGL2) | 粒子系统、泡沫、频谱 |
| 动画 | GSAP + Framer Motion | 弹簧物理、贝塞尔缓动 |
| 音频 | Web Audio API | 频谱分析、音频律动 |
| 登录 | Electron Session Partition | QQ音乐 + 网易云 OAuth |
| 后端 | Node.js (Express) | API代理、歌单获取 |
| 构建 | electron-builder | DMG 打包 |

### 1.4 目标平台
- macOS (arm64 优先, 兼容 x64)
- 窗口固定大小: 1200×800 (默认16:9变体)
- 支持桌面歌词独立窗口

## 2. 视觉设计规范

### 2.1 整体设计风格
- **风格**: 流体 (Fluid) — 半透明磨砂玻璃 / 动态水波纹 / 色差折射 / 多层模糊
- **配色**: 用户可在DIY中自定义，默认深色基底 + 动态流体色彩
- **动画原则**: 所有UI状态变化必须有弹簧动画过渡，禁止硬切
- **贝塞尔预设**:
  - 展开: `cubic-bezier(0.1, 1.1, 0.1, 1.1)` (弹性过冲)
  - 收起: `cubic-bezier(0.3, -0.3, 0, 1)` (轻微回弹)
  - 通用: `cubic-bezier(0.1, 0.9, 0.2, 1.0)`

### 2.2 层级结构 (Z轴)
```
Layer 0 — 流体动态背景 (全屏实时水波纹Shader)
Layer 1 — 穿透层 (透明, `pointer-events: none`, 除悬浮仓外全部穿透)
Layer 2 — 顶部气泡仓 (账号管理 + 3D歌单队列)  ← 鼠标可交互
Layer 3 — 左侧气泡仓 (歌单列表)              ← 鼠标可交互
Layer 4 — 右侧气泡仓 (歌词显示)              ← 鼠标可交互
Layer 4.5 — 底部控制器 (粒子沙感UI)          ← 鼠标可交互
Layer 5 — 中心核心布局 (歌曲粒子封面+信息+泡沫) ← 鼠标可交互
Layer 6 — DIY弹窗 (居中覆盖)                ← 鼠标可交互
```

## 3. 功能模块详述

### 3.1 中心核心布局 (Center Core)

**位置**: 窗口正中偏上，固定大小约 400×500 像素区域
**边框**: 极窄半透明边框 (`1px solid rgba(255,255,255,0.08)`)
**内容由上到下**:

#### 3.1.1 歌曲粒子封面 (Album Art Particles)
- **技术**: Three.js WebGL2 粒子系统，参考 Mineradio-MacOS `buildCoverParticleGeometry()` 
- **粒子规格**: 
  - 网格密度: 默认 118×118 分辨率 (可调 60~200)
  - 粒子大小: ~2-4px，圆形点精灵
  - 粒子形态: 如沙般细腻，1-2px 颗粒感
- **行为**:
  - 鼠标/触控拖动可旋转3D视角 (轨道旋转)
  - 随音频律动: 低/中/高三个频率层分别驱动不同深度层粒子起伏
  - 入场动画: 粒子从散落状态聚合成封面图像 (快速入场)
  - 切歌动画: 粒子如沙般散开 → 新封面聚合
- **多层渲染**: 至少3层粒子，分别响应 Bass / Mid / Treble 频率
- **演示图片**: 使用 Mineradio-MacOS `build/icon.png`

#### 3.1.2 歌曲名称 + 作者 (3D Text)
- 位于粒子封面下方
- 歌曲名: 大号字体, 3D环绕质感 (text-shadow 多层)
- 作者名: 稍小字体, 分行显示
- 动画: 切歌时文字3D翻转切换

#### 3.1.3 泡沫/棉花糖律动元素 (Foam Element)
- **技术**: Three.js 球体群 + 自定义 Shader, 参考 Mineradio-MacOS "泡沫 · 珍珠虹彩 · 柔波浮沉"
- **位置**: 歌曲名称下方
- **形态**: 多层球体粒子群，随不同音频频率上下起伏 + 旋转
- **配色**: 柔和高级感，随机变换 (珍珠白/虹彩/柔和粉蓝渐变)
- **质感**: 半透明 + 珍珠光泽 + 柔波浮沉动画
- **律动**: 多个频率层分别驱动不同球体群的 Y 轴位移和缩放

#### 3.1.4 设置按钮
- 位于中心核心布局右上角
- 点击弹出 DIY 设置弹窗

### 3.2 左侧气泡仓 — 歌单列表 (Playlist Chamber)

**触发逻辑**:
- 默认隐藏
- 鼠标悬浮中心核心左边缘区域时弹出 (带弹簧动画)
- 鼠标点击左边缘可常驻显示
- 再次点击边缘或点击仓外关闭

**视觉**:
- 气泡形态: 毛玻璃圆角容器，与中心核心保持间距 (≥24px)
- 居中对称于中心核心
- 内容: 歌单列表，当前播放高亮
- 滚动列表，项目hover时微动反馈

### 3.3 右侧气泡仓 — 歌词显示 (Lyrics Chamber)

**触发逻辑**:
- 默认隐藏
- 鼠标悬浮中心核心右边缘区域时弹出
- 点击右边缘可常驻显示

**视觉**:
- 气泡形态: 毛玻璃圆角容器，与中心核心保持间距 (≥24px)
- 居中对称
- 内容: 歌词逐行显示，当前句高亮
- 上下越远淡化 (CSS mask-image gradient)
- 可设置显示行数 (DIY中配置, 默认全部, 滑块0~N)

### 3.4 顶部气泡仓 — 账号 + 3D歌单队列 (Account & Queue Chamber)

**触发逻辑**:
- 默认显示在中心核心上方
- 靠近但不紧贴中心核心
- 悬浮时仓内元素有迎合动画

**内容**:
- **左侧**: 多平台账号登录状态 (QQ音乐 / 网易云 / 酷狗)
  - 登录按钮 → 触发 Electron OAuth 弹窗
  - 已登录显示头像
  - 支持多端同时登录
- **右侧**: 3D歌曲队列
  - 歌曲照片横向排列 (按播放顺序)
  - 中间高亮凸显当前播放歌曲 (放大)
  - 两端渐行渐远、越远越小
  - 3D透视旋转效果
  - 鼠标移动有生动的迎合动作

### 3.5 底部音乐控制器 (Music Controller)

**触发逻辑**: 始终可见，位于中心核心下方

**形态**: 
- 粒子沙感UI — 控制区由微小粒子组成
- 操作时: 周围粒子如沙被压下然后缓慢恢复
- 粒子颜色随音乐律动变化

**按钮** (参考 Mineradio-MacOS 按钮设计):
- 上一曲 / 播放暂停 / 下一曲
- 音量控制
- 播放模式 (顺序/随机/单曲循环)
- 进度条 (沙感粒子进度条)

**频谱可视化**:
- 多层粒子状频谱 (非传统柱状)
- 3D环绕排列
- 鼠标可拖动旋转视角
- 多层分别响应不同频率

### 3.6 DIY 设置弹窗 (Settings Overlay)

**触发**: 点击中心核心右上角设置按钮
**位置**: 居中覆盖在核心区域上方
**关闭**: 点击弹窗外区域或关闭按钮

**Tab 分类**:
| Tab | 内容 |
|-----|------|
| 粒子封面 | 粒子分辨率、散落强度、律动灵敏度、旋转速度、色彩 |
| 泡沫特效 | 泡沫数量、大小、虹彩强度、浮沉幅度、配色方案 |
| 歌词设置 | 显示行数(滑块0~N)、字体大小、颜色、高亮色、淡化强度 |
| 歌单设置 | 列表样式、字体、透明度 |
| 频谱设置 | 粒子层数、频率响应范围、3D旋转 |
| 控制器 | 粒子密度、沙感强度、按钮样式 |
| 背景流体 | 水波纹强度、速度、配色 |
| 气泡仓 | 各仓透明度、触发灵敏度、常驻/隐藏 |
| 账号 | 登录管理、多端切换 |

## 4. 音乐平台集成

### 4.1 登录授权
- **QQ音乐**: Electron session partition `persist:fluidmusic-qqmusic-login`, OAuth URL
- **网易云音乐**: Electron session partition `persist:fluidmusic-netease-login`, OAuth URL
- **酷狗**: 可选扩展
- **多端同时登录**: UI 支持显示多个平台登录状态

### 4.2 歌单获取
- 各平台 API 代理 (Node.js Express 后端)
- 获取用户歌单列表
- 获取歌单详情及歌曲 URL

### 4.3 音频播放
- Web Audio API 解码播放
- 实时频谱分析 (AnalyserNode, fftSize=2048)
- 分离 Bass(20-250Hz) / Mid(250-4kHz) / Treble(4k-20kHz) 频段

## 5. 动画系统规范

### 5.1 粒子封面动画
- 入场: 散落→聚合, 持续时间 1.2s, ease-out
- 切歌: 散开 0.4s → 聚合 0.8s
- 律动: 实时, 3层频率驱动

### 5.2 气泡仓动画
- 显示: scale(0.85→1.0) + opacity(0→1), 弹簧 0.6s
- 隐藏: scale(1.0→0.9) + opacity(1→0), 0.3s
- 常驻/临时状态切换: 边框亮度变化

### 5.3 泡沫动画
- 浮沉: 正弦波 Y 轴位移, 多频率叠加
- 旋转: Z轴连续旋转, 速度随 Mid 频率变化
- 配色: 每30s随机渐变切换

### 5.4 控制器粒子沙感
- 粒子压缩/恢复: 弹簧物理模拟
- 按钮hover: 粒子向按钮中心聚拢

## 6. 交互细节

### 6.1 边缘触发热区
- 中心核心四周边缘往外延伸 ~30px 为触发热区
- 热区颜色/光标: 默认不可见, hover时 `cursor: pointer`
- 四个边缘分别触发: 上(歌单队列)/左(歌单)/右(歌词)/下(控制器始终可见)

### 6.2 穿透层设计
- Layer 1 设置 `pointer-events: none`
- 气泡仓和核心区单独设置 `pointer-events: auto`
- 其他区域鼠标事件穿透到桌面

### 6.3 窗口行为
- 应用窗口固定大小 (不可改变)
- 支持 macOS 全屏模式
- 桌面歌词独立窗口 (可拖动、可调整大小)
- 壁纸模式窗口

## 7. DIY 配置持久化

### 7.1 存储方案
- 所有DIY配置保存至本地 JSON (`electron-store` 或 `localStorage`)
- 启动时自动加载配置
- **上次选择的特效预设 (柱形/泡沫/不规则) 必须持久化记忆**

### 7.2 默认配置
| 配置项 | 默认值 |
|--------|--------|
| 歌词显示行数 | 全部 (0表示不显示) |
| 粒子分辨率 | 118 |
| 泡沫数量 | 50 |
| 频谱层数 | 3 |
| 控制器样式 | 默认 |
| 背景流体强度 | 0.8 |
| 气泡仓透明度 | 0.15 |
